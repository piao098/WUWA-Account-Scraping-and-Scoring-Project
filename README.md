# 螃蟹鸣潮账号筛选推送器

一个用于筛选螃蟹平台鸣潮账号的桌面工具。它会从最新上架账号开始抓取，解析角色、武器、资源、饰品和交易展示信息，按可调整的评分规则计算账号价值，并生成报告或推送到手机。

## 项目描述

这个项目面向螃蟹平台上的鸣潮账号交易场景，把原本需要人每天蹲新号、翻商品、看截图、比角色命座和专武的过程，整理成一个可以持续运行的桌面工具。用户设定价格范围、发布时间、想要的角色或武器，以及自己的评分偏好后，程序会自动从螃蟹最新上架账号开始抓取，解析鸣潮账号里的角色、武器、抽卡资源和饰品，计算总分与性价比，并按价格段筛出更值得关注的候选。相比人工每天反复刷新和手动对比，程序可以定时爬取、自动评分、生成报告，并把高价值账号推送到手机。推送内容集中展示命座专武、成型队、等效抽数、热度和风险信息，让蹲号从临时人工判断变成可复用、可调整、可持续监控的流程。

## 项目效果

- 自动抓取指定价格和发布时间范围内的在售账号。
- 默认过滤 B 服，支持按角色、武器、五星数量、等效抽数、热度等条件筛选。
- 评分结果按总分、性价比和价格段 Top 展示。
- 推送内容会合并角色命座和专武，例如 `绯雪6+1`，并按成型队优先展示。
- 交易与风险信息不进入评分，但会在推送中辅助人工判断。
- 桌面端提供运行进度、定时推送、角色排行榜、公式编辑和推送预览。
- 所有筛选条件和评分参数都会保存到 `configs/`，关闭后再打开仍然保留。

## 快速开始

需要 Windows 和 Python。依赖管理建议使用 `uv`。

```powershell
uv venv .venv
uv run python desktop_app.py
```

如果本机已经能直接运行 Python，也可以双击：

```text
run_desktop_app.bat
```

## 推送配置

不同用户的手机推送配置不同，推荐直接在桌面程序的 `推送配置` 页面填写并保存。配置会写入本机 `notify_config.json`。

也可以手动复制示例配置：

```powershell
Copy-Item notify_config.example.json notify_config.json
```

然后把 `notify_config.json` 里的 `pushplus_token` 改成自己的 PushPlus token。`pushplus_topic` 可选，个人推送可以留空。

`notify_config.json` 已被 `.gitignore` 排除，不要提交真实 token。

## 主要页面

- `运行进度`：显示当前阶段、已用时间、预计剩余、列表账号数、详情进度。
- `筛选运行`：设置价格、发布时间、最低分、必备角色/武器等条件。
- `推送配置`：填写每个用户自己的 PushPlus token、topic 和通知标题。
- `定时推送`：程序保持打开时，可按间隔分钟或每天固定时间自动运行。
- `角色排行榜`：调整角色梯度、倍率、版本、定位和专武。
- `公式参数`：直接在公式中修改命座、专武、配队、资源、饰品和性价比参数。

## 默认策略

- 价格范围：`200-2000`
- 发布时间：最近 `7` 天
- 每个 `100` 元价格段展示前 `5` 个
- 价格和时间范围内的账号会全部抓详情，不按展示数量提前截断
- B 服直接过滤
- 黄数、交易热度、绑定风险不进入评分

## 评分口径

角色分：

```text
角色分 = 角色梯度值 * (1 + 命座分 + 专武分 + 配队命中分)
```

武器分只计算“不是任何已持有计分角色专武”的五星武器。已持有角色的专武已经并入角色分，不会再重复进入武器分。

资源分会把星声、月相、浮金波纹、余波珊瑚折算为等效限定抽；铸潮波纹不计分。

性价比会综合账号总分、价格对应期望分和价格效率，具体参数可在界面中修改。

## 打包 EXE

```powershell
.\build_exe.ps1
```

生成结果位于：

```text
dist/
```

如果分享给别人，发送整个 `dist` 文件夹即可。对方需要放入自己的 `notify_config.json` 才能正式推送。
